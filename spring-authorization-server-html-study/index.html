这本小书适合谁？

适合已经会 Spring Boot 3、Spring Security、微服务开发，但对 OAuth2 / OIDC 授权服务器还没有形成完整系统图的 Java 后端。

这里默认你不是为了写一个简单 login 接口，而是准备新增独立的 conference-module-auth，让它成为整个 conference 系统的 OAuth2 / OIDC 鉴权中心。

官方依据： 本项目固定基于 Spring Authorization Server 1.5.8 Reference。 官方 Overview 说明 Spring Authorization Server 是构建在 Spring Security 之上的 OAuth2 / OpenID Connect 1.0 授权服务器框架。 本项目中的 conference 架构拆分、Gateway 边界、Feign 服务 Token 方案，属于项目落地解释，不是官方原文。

conference 项目目标架构

你要做的不是“system 模块继续登录，然后顺手发个 JWT”。 更清晰的边界是：auth 专心发 Token，system 专心管用户和权限档案，Gateway 专心守入口，业务服务专心处理业务。

推荐阅读路线

  1. 第 0～2 章：先把脑图搭起来，不急着写配置。
  2. 第 3～6 章：再看最小可运行配置、客户端注册、Token 与 JWK。
  3. 第 7～8 章：把浏览器登录和服务间调用分别跑通。
  4. 第 9～11 章：进入生产化：持久化、权限 Claims、Gateway、密钥、审计和安全清单。
先记住： Spring Authorization Server 不是“加了 JWT 的登录接口”。 它是 OAuth2 / OIDC 授权服务器框架。你要先确定系统边界，再写配置。
